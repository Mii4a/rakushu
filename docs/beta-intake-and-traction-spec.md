# らくしゅう β導線・初期トラクション計測仕様

## 目的
LP 訪問から β 参加登録までの初期反応を計測し、需要の兆しと guided private beta の運用判断を見えるようにする。

## βの位置づけ
- いまは public beta ではなく guided private beta
- 完成版のセルフサーブ提供ではなく、少人数を順番に案内しながら改善する段階
- 反応を見るだけでなく、期待値のズレと対象外入力を早く見抜くことも目的に含める

## 最初に置く CTA
- 文言案 A: 無料でβ参加して、求人票の見極めを先に試す
- 文言案 B: ブラック求人を避けたい人向けβ版に参加する
- 文言案 C: 求人票の不安を減らす新機能を先に試す

## βフォームの最小入力項目
必須:
- 連絡先（メール or Discord）
- 現在の状況（新卒就活 / 転職活動 / 情報収集中）
- いま一番困っていること

任意:
- 志望職種
- 1 週間で見る求人本数
- ヒアリング協力可否

## 初期βで受ける入力の考え方
### 強く受けたい
- 求人本文がまとまっている
- 条件比較に必要な情報が本文にある程度含まれる
- 比較・見極め・応募整理の困りごとが明確

### 条件つきで受ける
- teaser / prose 寄りで情報が薄い
- 別ページ依存が強い
- `本文未記載` や `要確認` が増えうることを先に伝える

### 深追いしない
- 本文自体に比較材料がほぼない
- OCR や外部資料がないと判断材料が出ない
- 1件の求人ではなく採用LP全体しかない

## 最初のKPI
- LP訪問数
- CTAクリック率
- βフォーム到達率
- βフォーム送信率
- ヒアリング協力率
- デモ開始率
- 解析完了率
- 保存率

## 週次で見る追加指標
- thin-input 由来の相談件数
- parser miss 疑い件数
- `本文未記載 / 要確認` への戸惑い件数
- 困りごとカテゴリ別の送信件数

## 最初の判断基準
- LP訪問はあるのに CTA が押されない
  - 訴求が弱い or ターゲットがずれている
- CTA は押されるのに送信されない
  - フォーム負荷が高い or 不安が勝っている
- 送信はあるがデモ利用につながらない
  - ソリューションのつながりが弱い
- ヒアリング協力率が高い
  - 課題の強さが高い可能性
- `本文未記載 / 要確認` への違和感が多い
  - 期待値調整か説明UIが足りない可能性

## 運用上の見る順番
1. expectation mismatch で説明できるか
2. input scope の外か
3. parser miss か
4. rubric / UI copy の不足か

## まず実装しなくてよいもの
- 複雑なスコアリングダッシュボード
- 高度なセグメント分析
- 大量のフォーム項目
- 自動メール育成

## 実装メモ
- はじめは 1 本の β 導線でよい
- CTA 文言は 2〜3 種まで
- まずは「連絡先を残すかどうか」を最重要シグナルにする
- 定量だけでなく、自由記述の悩み文面を必ず残す
- 少人数案内制であることをコピーと送信後メッセージの両方で揃える
