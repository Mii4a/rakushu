# Beta Launch Checklist

## Hard gate（満たさないと募集しない）
- [ ] `npm test` が green
- [ ] `npm run build` が green
- [ ] βページに「案内制β」「改善中」「完全対応ではない」が明記されている
- [ ] βフォーム送信後メッセージが順番案内の期待値になっている
- [ ] `本文未記載 / 要確認` の説明が主要画面にある
- [ ] 初期βの対象入力ルールが docs 化されている
- [ ] βフィードバックの週次レビュー手順が docs 化されている

## Soft gate（あると安全）
- [ ] 向いている人 / 向いていない人の案内がある
- [ ] thin-input を parser bug と切り分ける運営メモがある
- [ ] launch後に毎週見る KPI が定義されている
- [ ] ヒアリング対象の選び方が決まっている
- [ ] main push で本番デプロイされる前提の確認が取れている

## No-go sign（今は募集しない）
- [ ] βなのに完成品と誤認されるコピーしかない
- [ ] thin-input と parser miss の切り分け方がない
- [ ] 保存後画面で missing-item 表示の意味が伝わらない
- [ ] フォーム送信後の次アクションが曖昧
- [ ] 同一媒体で再現する重大崩れが未整理

## Launch day quick check
1. βページを開く
2. 期待値調整コピーが見えることを確認
3. フォーム送信が通ることを確認
4. 成功メッセージが順番案内になっていることを確認
5. 求人入力画面で missing-item の説明が見えることを確認
6. 保存済み求人画面で same copy が見えることを確認

## Launch後 1週間で見ること
- どの悩みカテゴリが多いか
- 途中離脱がどこで起きるか
- `本文未記載 / 要確認` に対する戸惑いが出ていないか
- thin-input を parser bug と誤認した声が多くないか
